This guide walks you through obtaining an API key, writing a minimal C program that fetches stock price data, and compiling and running it.

## Before you begin

Make sure you have [installed the Axion C SDK](/installation) and can compile against it.

<Steps>
  <Step title="Get an API key">
    Log in to the [Axion dashboard](https://axionquant.com/dashboard/api-keys) and create a new API key. Copy the key — you will pass it to `axion_init()` at runtime.

    <Warning>
      Keep your API key out of source control. Use an environment variable or a secrets manager rather than hardcoding the value in your source.
    </Warning>
  </Step>

  <Step title="Write the program">
    Create a file named `main.c` with the following content:

    ```c main.c
    #include <stdio.h>
    #include <stdlib.h>
    #include <axion.h>

    int main(void) {
        const char *api_key = getenv("AXION_API_KEY");
        if (!api_key) {
            fprintf(stderr, "Error: AXION_API_KEY environment variable not set\n");
            return 1;
        }

        /* Initialize the client with your API key */
        AxionClient *client = axion_init(api_key);
        if (!client) {
            fprintf(stderr, "Error: failed to initialize Axion client\n");
            return 1;
        }

        /* Fetch stock price data for AAPL */
        AxionResponse *response = axion_stocks_prices(client, "AAPL", "2024-01-01", "2024-12-31", "1d");
        if (!response) {
            fprintf(stderr, "Error: no response returned\n");
            axion_free_client(client);
            return 1;
        }

        if (response->error) {
            fprintf(stderr, "API error: %s\n", response->error);
        } else {
            printf("HTTP status : %d\n", response->http_status);
            printf("Response    : %s\n", response->data);
        }

        /* Always free the response and client when done */
        axion_free_response(response);
        axion_free_client(client);

        return 0;
    }
    ```

    The program uses three core SDK functions:

    | Function | Purpose |
    |---|---|
    | `axion_init(api_key)` | Allocates and configures an `AxionClient` |
    | `axion_stocks_prices(client, ticker, from, to, frame)` | Fetches historical price data for a stock ticker |
    | `axion_free_response(response)` | Releases memory owned by the `AxionResponse` |
    | `axion_free_client(client)` | Tears down the client and releases its resources |
  </Step>

  <Step title="Compile the program">
    <CodeGroup>

    ```bash Static linking
    gcc -Ipath/to/axion-c-sdk/include \
        -o main main.c \
        path/to/axion-c-sdk/libaxion.a \
        -lcurl
    ```

    ```bash Dynamic linking (system-wide install)
    gcc -o main main.c -laxion -lcurl
    ```

    </CodeGroup>
  </Step>

  <Step title="Run the program">
    Export your API key and run the binary:

    ```bash
    export AXION_API_KEY="your_api_key_here"
    ./main
    ```

    A successful response looks like:

    ```
    HTTP status : 200
    Response    : {"symbol":"AAPL","price":189.42,"currency":"USD","timestamp":1742400000}
    ```

    <Tip>
      The raw JSON string is available in `response->data`. If the response was parsed successfully, `response->json` holds a `cJSON *` pointer you can traverse with the cJSON API.
    </Tip>
  </Step>
</Steps>

## Error handling

Always check both `response` itself and `response->error` before accessing `response->data` or `response->json`:

```c
if (!response) {
    /* Network-level failure or allocation error */
}
if (response->error) {
    /* API returned an error; message is in response->error */
}
/* Safe to use response->data and response->json here */
```

## Memory ownership

- `axion_init()` allocates the client — always pair it with `axion_free_client()`.
- Every `AxionResponse *` returned by the SDK is heap-allocated — always pair it with `axion_free_response()`.
- Do not access `response->data`, `response->json`, or `response->error` after calling `axion_free_response()`.

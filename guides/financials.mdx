---
title: "Financials"
description: "Access income statement, balance sheet, cash flow, share data, and financial snapshots for any public company."
---

The Axion C SDK provides granular financial data functions grouped by statement type. Every per-period function accepts a `periods` argument that controls how many historical periods are returned.

## The `periods` parameter

All per-period functions share this signature pattern:

```c
AxionResponse *axion_financials_<metric>(
    AxionClient *client,
    const char  *ticker,
    int          periods   // 0 = default, positive integer = specific number of periods
);
```

| `periods` value | Behaviour |
|---|---|
| `0` | Returns the default number of periods (typically the most recent quarters or years) |
| Positive integer | Returns exactly that many periods, most recent first |

<Tip>
  Start with `0` to see the default response shape, then switch to a specific integer once you know how many periods your application needs.
</Tip>

## Income statement

```c
AxionResponse *axion_financials_revenue(client, ticker, periods);
AxionResponse *axion_financials_net_income(client, ticker, periods);
```

- **revenue** — Total revenue across all reported periods.
- **net_income** — Net income (profit after all expenses and taxes).

## Balance sheet

```c
AxionResponse *axion_financials_total_assets(client, ticker, periods);
AxionResponse *axion_financials_total_liabilities(client, ticker, periods);
AxionResponse *axion_financials_stockholders_equity(client, ticker, periods);
AxionResponse *axion_financials_current_assets(client, ticker, periods);
AxionResponse *axion_financials_current_liabilities(client, ticker, periods);
```

- **total_assets** — Sum of all assets on the balance sheet.
- **total_liabilities** — Sum of all liabilities.
- **stockholders_equity** — Total equity (assets minus liabilities).
- **current_assets** — Assets expected to be converted to cash within one year.
- **current_liabilities** — Obligations due within one year.

## Cash flow statement

```c
AxionResponse *axion_financials_operating_cash_flow(client, ticker, periods);
AxionResponse *axion_financials_capital_expenditures(client, ticker, periods);
AxionResponse *axion_financials_free_cash_flow(client, ticker, periods);
```

- **operating_cash_flow** — Cash generated by core business operations.
- **capital_expenditures** — Cash spent on property, plant, and equipment.
- **free_cash_flow** — Operating cash flow minus capital expenditures.

## Share data

```c
AxionResponse *axion_financials_shares_outstanding_basic(client, ticker, periods);
AxionResponse *axion_financials_shares_outstanding_diluted(client, ticker, periods);
```

- **shares_outstanding_basic** — Common shares currently outstanding.
- **shares_outstanding_diluted** — Shares outstanding including all dilutive securities (options, warrants, convertible notes).

## Comprehensive views

Two functions return multi-metric snapshots without a `periods` argument:

```c
AxionResponse *axion_financials_metrics(AxionClient *client, const char *ticker);
AxionResponse *axion_financials_snapshot(AxionClient *client, const char *ticker);
```

- **metrics** — A collection of derived financial ratios and per-share metrics (EPS, P/E, P/FCF, debt-to-equity, and similar).
- **snapshot** — A broad cross-statement view combining the most recent values from the income statement, balance sheet, and cash flow statement in a single response.

<Info>
  `axion_financials_metrics` and `axion_financials_snapshot` do not accept a `periods` parameter — they always return the latest available data.
</Info>

## Example: revenue and free cash flow for AAPL

The following example fetches the last 8 quarters of revenue and free cash flow for Apple, then prints both payloads.

```c
#include "axion.h"
#include <stdio.h>

int main(void) {
    AxionClient *client = axion_client_new("YOUR_API_KEY");
    if (!client) {
        fprintf(stderr, "Failed to create client\n");
        return 1;
    }

    // Last 8 quarters of revenue
    AxionResponse *revenue = axion_financials_revenue(client, "AAPL", 8);
    if (revenue->error) {
        fprintf(stderr, "Revenue error: %s\n", revenue->error);
    } else {
        printf("Revenue (HTTP %d):\n%s\n\n", revenue->http_status, revenue->data);
    }
    axion_response(revenue);

    // Last 8 quarters of free cash flow
    AxionResponse *fcf = axion_financials_free_cash_flow(client, "AAPL", 8);
    if (fcf->error) {
        fprintf(stderr, "Free cash flow error: %s\n", fcf->error);
    } else {
        printf("Free Cash Flow (HTTP %d):\n%s\n", fcf->http_status, fcf->data);
    }
    axion_response(fcf);

    axion_client(client);
    return 0;
}
```

## Example: financial snapshot

```c
#include "axion.h"
#include <stdio.h>

int main(void) {
    AxionClient *client = axion_client_new("YOUR_API_KEY");
    if (!client) return 1;

    AxionResponse *snap = axion_financials_snapshot(client, "AAPL");
    if (snap->error) {
        fprintf(stderr, "Snapshot error: %s\n", snap->error);
    } else {
        printf("%s\n", snap->data);
    }

    axion_response(snap);
    axion_client(client);
    return 0;
}
```

## Memory management

```c
axion_response(resp);   // free AxionResponse
axion_client(client);   // free AxionClient
```

<Warning>
  Call `axion_response` on every `AxionResponse` pointer you receive, including error responses. Skipping this call leaks the `data`, `json`, and `error` fields.
</Warning>
